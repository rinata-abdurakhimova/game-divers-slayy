# Jam Brainstorm

Read `docs/SCOPE.md`, `OWNERSHIP.md`, and any current concept notes.

1. Interpret the theme literally, mechanically, and emotionally.
2. Produce exactly three concepts small enough for a short Godot jam.
3. For each, state the core mechanic, hook, theme connection, implementation risk, polish potential,
   three-member split, and smallest playable version.
4. Recommend one concept based on integration risk and presentation clarity.
5. Define must/should/could/cut scope and a vertical-slice test.
6. Update `docs/SCOPE.md` when the team selects a concept.

Do not write implementation code unless explicitly requested. Prefer one strong mechanic, quick
restart, visible feedback, and a demoable theme connection.
